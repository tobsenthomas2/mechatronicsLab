    This is mechatronics, so let’s control a digital output pin.        pinC0 = pyb.Pin (pyb.Pin.board.PC0, pyb.Pin.OUT_PP)

       pinA1 = pyb.Pin (pyb.Pin.board.PA1, pyb.Pin.OUT_PP)

Control an Actual Thing

An efficient way to control the torque in a DC motor or brightness of an LED is pulse width modulation. Pulse width modulation uses a square wave of varying duty cycle (the fraction of time a signal is on) to modulate the average voltage. In the drawing below, a voltage of VM delivers full voltage to a motor and 0 volts delivers zero net voltage, so as the PWM duty cycle changes from about 40% to about 80%, it causes an increase in motor torque, assuming roughly constant motor speed.

Most microcontrollers contain hardware which, when configured properly, creates square waves with controllable duty cycles. Once the hardware has been configured correctly, you just have to tell it the percentage of time the signal should be on as a fraction (or percentage) of the wave period, and the hardware will keep sending that wave until told to do otherwise.

This week we’ll make a program which uses PWM to control the brightness of an LED. Next week we’ll use it to control a motor.

  • Connect an LED through a 220Ω – 330Ω resistor to the 3V3 pin and pin PA0 on the Nucleo. You can use the GND and 3V3 pins to check which way the LED and resistor must be connected so it lights (the LED is a diode, so only one way will work).

  • Set pin PA0 as an output (as above) and verify that you can turn the LED on and off.

  • Modify the PWM code from above to control the LED’s brightness. You need to look on the STM32L476RG Nucleo Arduino Pins diagram on Canvas to see which timer and channel must be used. The purple boxes showing PWMx/y numbers indicate that you must use Timer x and Channel y for a given pin.

    Because the LED will be on when the PWM signal is a logic zero, you must set the timer channel to mode pyb.Timer.PWM_INVERTED to control LED brightness.

  • Copy your code into Thonny’s editor window to make a script that you can run. Test your code by running it several times and verifying that it works. Save the file you’re writing in your src directory as main.py.

  • Write two functions called led_setup() and led_brightness() in your file. The setup function should prepare the PWM timer and channel, and the brightness function should set the PWM duty cycle.

    Each function should be commented with Doxygen style comments. See the file the_answer.py on Canvas for an example of how your Doxygen comments should be formatted.

  • Write a script in main.py which calls the functions repeatedly in a loop to make the LED’s brightness slowly increase from 0 to maximum over 5 seconds, then go back to 0 and repeat the process indefinitely until someone cuts off the Nucleo’s power or presses Ctrl-C. The script should be below the functions, with the file following the format below:

       """! Doxygen style docstring for the file """

   def led_setup ():
       """! Doxygen style docstring for this function """
       Function code here

   def led_brightness ():
       """! Doxygen style docstring for this function """
       More function code here

   if __name__ == "__main__":
       Script code goes here

    The example file the_answer.py on Canvas shows a more detailed example of how the file should be organized.
